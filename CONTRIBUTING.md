# 贡献指南
欢迎各位同学为本项目提交代码、修复bug、优化文档！

## 一、开发环境要求
1. Python 3.8+
2. 遵循项目原有代码缩进规范（4空格缩进）
3. 新增代码必须配套单元测试，放置在tests文件夹

## 二、提交代码流程
1. 拉取main分支最新代码，保证本地代码同步
2. 新建分支开发，分支命名规范：
   - 修复bug：fix/xxx
   - 新增功能：feature/xxx
   - 文档修改：docs/xxx
3. 开发完成后本地运行全部单元测试，无报错再提交
4. 提交信息简洁清晰，说明本次修改内容
5. 提交分支后发起Pull Request，等待小组审核合并

## 三、代码规范
1. 变量、函数使用小写下划线命名
2. 关键逻辑添加注释说明用途
3. 禁止提交无效测试代码、打印调试语句
4. 新增功能需同步更新README使用说明

## 四、文档规范
修改README、CONTRIBUTING等文档时，保证Markdown格式统一，代码块完整闭合，标题排版整齐。

## 五、提问与反馈
如有需求、bug、优化建议，可以在仓库Issues板块提交，标注对应模块名称。

## 六、代码提交PR完整操作步骤
1. 打开项目主仓库页面，点击右上角Fork，将仓库复制到自己的GitHub账号
2. 在自己Fork后的仓库内新建、修改对应代码/文档文件
3. 顶部导航栏切换至Pull Requests，点击New Pull Request
4. 源分支选择你个人Fork仓库的对应分支，目标分支选择主仓库main分支
5. 填写清晰的提交说明，点击Create Pull Request发起合并申请
6. 等待组长审核代码/文档，审核通过后完成合并
